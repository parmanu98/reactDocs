React Folder Structure
  • my-app/
    • node_modules/

      Contains all npm packages and dependencies.

    • public/
      • index.html – Main HTML template.
      • favicon.ico – Browser tab icon.
      • manifest.json – PWA configuration.
      • robots.txt – Crawlers instructions.
      • assets/ – Images, fonts, icons.
    • src/
      • App.js – Main app component.
      • App.css – Styles for App.js.
      • index.js – Entry point.
      • index.css – Global CSS.
      • logo.svg – React logo.
      • reportWebVitals.js – Performance metrics.
      • setupTests.js – Test setup.
      • components/
        • Button/
          • Button.js – Component code
          • Button.css – Component styles
          • Button.test.js – Component tests
      • pages/
        • Home.js – Home page
        • About.js – About page
        • Contact.js – Contact page
      • services/
        • api.js – Axios/fetch setup
        • auth.js – Auth API calls
        • user.js – User API calls
      • hooks/
        • useAuth.js – Authentication hook
        • useFetch.js – Fetch hook
        • useLocalStorage.js – LocalStorage hook
      • contexts/
        • AuthContext.js – Auth state
        • ThemeContext.js – Theme state
      • utils/
        • validators.js – Validation helpers
        • formatDate.js – Date helpers
        • constants.js – App constants
    • .gitignore
    • package.json
    • package-lock.json / yarn.lock
    • README.md
    • .env
React Commands
  • Create React App

    npx create-react-app my-app – Create a new React project.
    yarn create react-app my-app – Yarn alternative.

  • Start Development Server

    npm start or yarn start – Runs dev server with hot reload.

  • Build Project

    npm run build or yarn build – Build optimized production-ready code.

  • Run Tests

    npm test or yarn test – Run Jest tests in watch mode.

  • Install / Remove Packages

    npm install package-name / yarn add package-name – Install packages.
    npm uninstall package-name / yarn remove package-name – Remove packages.

  • Linting

    npm run lint – Check code style.
    npm run lint -- --fix – Automatically fix issues.

  • Serve Production

    npx serve -s build – Serve production build locally.

  • Eject CRA

    npm run eject – Exposes configuration files. Irreversible.

React Workflow Tips & Best Practices
  • Use **component-first approach**: Keep reusable components separate from pages.
  • Use **feature-based grouping** for large apps (each feature has components, services, and styles).
  • Keep **src clean**; avoid dumping files at top-level.
  • Use **PascalCase** for components, **camelCase** for hooks/utilities.
  • Keep CSS close to components using **CSS Modules, SCSS, or Tailwind classes**.
  • Use **Context API or Redux** for global state management instead of prop drilling.
  • Write **unit tests** for components and custom hooks.
  • Use **eslint + prettier** for consistent code formatting.
