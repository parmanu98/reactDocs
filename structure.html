React.js Folder Structure

Detailed explanation of every folder and file

Root of a React Project

my-app/
├── node_modules/
├── public/
├── src/
├── .gitignore
├── package.json
├── README.md
├── yarn.lock / package-lock.json
└── .env
  • node_modules/: Contains all npm packages and dependencies installed. Never edit manually; generated automatically.
  • public/: Stores static files like index.html, favicon, and assets that are served directly without being processed by Webpack.
  • src/: The main application code – all React components, pages, hooks, and styles reside here.
  • .gitignore: Specifies files and folders Git should ignore.
  • package.json: Contains project metadata, scripts, and dependencies.
  • README.md: Documentation for the project.
  • yarn.lock / package-lock.json: Locks package versions for reproducible builds.
  • .env: Environment variables configuration.

public/ Folder

public/
├── index.html
├── favicon.ico
├── manifest.json
└── assets/
  • index.html: The main HTML template where React app is mounted at <div id="root"></div>.
  • favicon.ico: Small icon shown in browser tabs.
  • manifest.json: Used for PWA configuration like app name, icons, and theme colors.
  • assets/: Optional folder to store images, fonts, or other static files. Can be used with process.env.PUBLIC_URL for access.

src/ Folder

src/
├── assets/
├── components/
├── pages/
├── services/
├── hooks/
├── contexts/
├── utils/
├── App.js
└── index.js
  • assets/: Stores static files imported into components, like:
    • images/ – PNG, JPG, SVG files
    • icons/ – App icons
    • fonts/ – Custom fonts
  • components/: Reusable UI components. Example structure: 
    Button/
├── Button.js
├── Button.css
└── Button.test.js
    Keep JS, CSS, and tests together for maintainability.
  • pages/: Components representing routes or screens: 
    pages/
├── Home.js
├── About.js
└── Contact.js
    Each page can use multiple smaller components.
  • services/: API calls and business logic: 
    services/
├── api.js      // Axios or fetch setup
├── auth.js     // Auth related API calls
└── user.js     // User API calls
    Keeps components clean and separates logic.
  • hooks/: Custom React hooks for reusability: 
    hooks/
├── useAuth.js
├── useFetch.js
└── useLocalStorage.js
  • contexts/: Global state management using React Context API: 
    contexts/
├── AuthContext.js
└── ThemeContext.js
    Wrap your <App /> with providers here.
  • utils/: Utility/helper functions: 
    utils/
├── validators.js
├── formatDate.js
└── constants.js
    Keep helper functions separate to avoid clutter in components.
  • App.js: Main component, usually handles routing and layout.
  • index.js: Entry point of the app. Renders <App /> to root.

Optional / Advanced Folders

src/
├── redux/       // Redux store and slices
├── routes/      // Route definitions
├── themes/      // Styled-components or Tailwind theme configs
└── config/      // Application configuration files

Large apps may use these folders for advanced state management, centralized routing, theming, or configuration.

Best Practices

  • Use component-first approach: keep reusable components separate from pages.
  • For larger projects, use feature-based grouping where each feature contains its components, services, and styles.
  • Keep src/ clean and organized; avoid dumping files at the top level.
  • Use PascalCase for components and camelCase for utils/hooks.
  • Keep CSS close to components using CSS modules, SCSS, or Tailwind classes.
